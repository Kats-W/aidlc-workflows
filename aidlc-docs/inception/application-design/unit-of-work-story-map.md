## 2. U-01 はストーリーなし（明記）

**U-01（Core Infrastructure）には直接対応するユーザーストーリーが存在しない。** これは意図的な設計判断であり、以下の理由による。

- U-01 は DynamoDB 全 5 テーブル・IAM・Secrets Manager・KMS・ログ・Connect インスタンス・Lex ボット骨格といった**全ユニット共有の基盤リソース**を提供する。
- ユーザー価値（ユーザーストーリー）は U-02〜U-07 の機能ユニットを通じて実現され、U-01 はその実現を可能にする土台である。
- U-01 の完了は機能要件ではなく**非機能要件（NFR）と Security Extension 全ルール準拠**で評価される（`unit-of-work.md` の U-01 DoD 参照）。

したがって U-01 は「ストーリー 0 件」が正常であり、ストーリー未割り当ての欠落ではない。

## 5. 実装優先順位（ユニット依存順 × ストーリー重要度）

依存順序（`unit-of-work.md` の確定シーケンス）を第一基準とし、各ユニット内ではコア機能・先行依存ストーリーを優先する。

| 順位 | Unit | 優先実装ストーリー（ユニット内順） | 優先理由 |
|---|---|---|---|
| 1 | **U-01** | （ストーリーなし） | 全ユニットの前提。Security Extension 全ルール準拠が完了条件。 |
| 2 | **U-02** | US-2.1 → US-2.2 → US-2.3 | RAG 検索のナレッジ基盤。クロール→差分→ベクトル化の自然順。U-03 が依存。 |
| 3 | **U-03** | US-1.1 / US-1.2（RAG コア）→ US-6.1 / US-6.2（履歴・パーソナライズ）→ US-1.3（エスカレーション判定）→ US-1.4（CSAT） | 一次解決率 70% の中核。8秒制約・PII マスクが集中。最重要ユニット。 |
| 4 | **U-04** | US-4.3（AI 復帰・CSAT）→ US-4.1（音声→チャット）→ US-4.2（チャット→音声） | U-03 完了直後に着手（U-05 より先行）。US-4.3 は U-03 の CSAT フローへ最短連結。 |
| 5 | **U-05** | US-5.2（プロファイル付与）→ US-5.1（チャット開始）→ US-6.3（CRM 書き込み） | U-03 のパーソナライズ前提を満たす US-5.2 を先行。CRM はスコープ外連携口のため後。 |
| 6 | **U-06** | US-3.1（低品質検出）→ US-3.2（ギャップ分析）→ US-3.3（提案生成） | 検出→分析→生成の直列依存。U-07 への提案データ供給元。 |
| 7 | **U-07** | US-7.3（認証）→ US-7.1（提案承認）→ US-7.2（統計） | 最後。認証ガードを先に確立し、U-06 の提案データを承認 UI で消費。 |

### 重要度（ビジネス価値）補足
- **最優先機能群**: US-1.1 / US-1.2 / US-6.2（一次解決率・パーソナライズ）— 成功基準「AI 解決率 ≥ 70%」「P99 ≤ 5 秒」に直結。
- **コンプライアンス必達**: US-6.1（PII マスク保存）・U-01 の Security Extension — 金融コンプライアンス要件で先送り不可。
- **段階的価値**: U-04/U-05（体験向上）→ U-06/U-07（運用改善サイクル）の順で価値が積み上がる。
